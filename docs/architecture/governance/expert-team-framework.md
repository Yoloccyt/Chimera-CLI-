# Chimera CLI 专家团队协作框架

> 版本: v2.2.0-omega | 日期: 2026-07-20 | 状态: Active

## 概述

本文档定义 Chimera CLI 项目的专家子代理团队结构、角色分工与协作流程。
基于 OMEGA 架构的 10 层设计，团队按专业领域分工，确保跨层协同高效。

## 团队结构

### E01 — 首席架构师
- **职责范围**: 架构决策、依赖合规审查、ADR 记录、技术选型
- **核心能力**: 
  - 10 层架构分层设计与依赖铁律（§2.2）审查
  - OMEGA 四定律（Ω-Sparse/Ω-Compress/Ω-Evolve/Ω-Event）合规性评估
  - 跨层设计决策与 ADR 编写
- **输出物**: 架构审查报告、ADR 文档、技术选型建议

### E02 — 全栈开发工程师（后端）
- **职责范围**: L1-L7 层（Core → Execution）代码实现与重构
- **核心能力**:
  - Rust 2021 + Tokio async 生态
  - EventBus 事件驱动架构
  - 库层 thiserror / 应用层 anyhow 错误分层
- **输出物**: 实现代码、单元测试、API 文档

### E03 — 全栈开发工程师（前端/接口）
- **职责范围**: L8-L10 层（Parliament → Interface）代码实现与重构
- **核心能力**:
  - TUI 开发（ratatui + crossterm）
  - MCP Mesh 跨进程通信
  - CHTC 跨平台适配
- **输出物**: TUI 面板代码、MCP 节点实现、跨平台兼容代码

### E04 — 测试专家
- **职责范围**: 测试策略、覆盖率审计、质量门禁
- **核心能力**:
  - TDD（RED-GREEN-REFACTOR）流程执行
  - proptest 属性测试设计
  - criterion benchmark 编写
  - OWASP A01-A10 安全测试
- **输出物**: 测试用例、覆盖率报告、质量门禁检查结果

### E05 — DevOps 工程师
- **职责范围**: CI/CD 验证、发布检查、Docker 镜像管理
- **核心能力**:
  - GitHub Actions workflow（audit/release/fuzz）
  - Docker 多阶段构建（distroless + nonroot）
  - cargo-audit 安全审计
  - 磁盘空间管理
- **输出物**: CI 配置、Dockerfile、发布检查清单结果

## 协作流程

```
需求分析 → 架构审查(E01) → 任务拆分(E01+E02+E03)
    → 并行实现(E02+E03) → 测试编写(E04)
    → 交叉审查(E01+E04) → 集成验证(E05)
    → 发布检查(E05) → 部署
```

### 阶段 1: 需求分析与架构审查
- E01 主导，产出架构审查报告与 ADR（如需）
- 确认变更与 OMEGA 四定律对齐

### 阶段 2: 任务拆分
- E01+E02+E03 协同，将需求拆分为 P0-P4 优先级任务
- 明确任务依赖关系与并行执行路径

### 阶段 3: 并行实现
- E02 负责 L1-L7 层变更，E03 负责 L8-L10 层变更
- 遵循 TDD 流程：先写失败测试再实现

### 阶段 4: 测试编写
- E04 为新增/修改代码编写测试
- 确保覆盖率达标（核心模块 ≥ 80%）

### 阶段 5: 交叉审查
- E01 审查架构合规性，E04 审查测试质量
- Critical 级问题阻塞合并

### 阶段 6: 集成验证
- E05 运行全量测试回归（cargo test/clippy/fmt/check）
- 验证 Docker 镜像构建与 --version 输出

### 阶段 7: 发布检查
- E05 执行 §7.2 发布前检查清单
- 通过后推送 tag 触发 CI/CD

## 沟通机制

- **每日站会**: 同步进度、识别阻塞、调整优先级
- **周度评审**: 审查测试覆盖率、技术债务、风险清单
- **月度验证**: 全量回归 + 安全审计 + 性能基准
