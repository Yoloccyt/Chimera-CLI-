# Chimera CLI 任务优先级量化体系

> 版本: v2.2.0-omega | 日期: 2026-07-20 | 状态: Active

## 概述

本文档定义 Chimera CLI 项目的任务优先级量化体系（P0-P4），
用于所有任务的排序、资源分配与执行调度。

## 优先级定义

### P0 — 阻断级（Blocker）
- **判定标准**:
  - 阻塞 CI/CD 流水线（cargo test/clippy/fmt/check 失败）
  - 安全漏洞（cargo audit 高危告警、OWASP 测试失败）
  - 数据丢失风险（内存泄漏、持久化损坏）
  - 依赖铁律违规（向上依赖）
- **响应时间**: 立即处理，阻塞所有其他任务
- **实例**: 
  - `#![forbid(unsafe_code)]` 缺失
  - Critical 安全事件丢失（SkepticVeto/RedTeamAudit）
  - 发布前检查清单失败

### P1 — 核心级（Critical）
- **判定标准**:
  - 核心功能缺陷（Quest 创建/执行/持久化失败）
  - 性能退化 > 20%（benchmark 回归）
  - 测试覆盖率下降（核心模块 < 80%）
  - OMEGA 四定律违规
- **响应时间**: 当前迭代内处理
- **实例**:
  - EventBus 事件丢失
  - HCW 窗口压缩失败
  - OSA 稀疏度 < 40%

### P2 — 优化级（High）
- **判定标准**:
  - 非核心功能增强（TUI 面板扩展、数据导出优化）
  - 性能优化 < 20%（O(n log n) → O(n) 改进）
  - 代码质量改进（clippy warning 修复）
- **响应时间**: 下一迭代内处理
- **实例**:
  - 动态 tick 模式实现
  - 批量操作支持
  - 过滤搜索功能

### P3 — 增强级（Medium）
- **判定标准**:
  - 用户体验改进（快捷键优化、状态栏增强）
  - 文档完善（API 文档、架构文档补充）
  - 工具链优化（构建脚本改进）
- **响应时间**: 按需排期
- **实例**:
  - 视图状态持久化
  - 专家团队框架文档
  - 磁盘清理脚本

### P4 — 维护级（Low）
- **判定标准**:
  - 技术债务偿还（废弃 API 清理、依赖升级）
  - 代码清理（dead code 移除、注释完善）
  - 非关键依赖更新
- **响应时间**: 按季度排期
- **实例**:
  - rust-toolchain 更新
  - 废弃 workspace dependency 清理
  - 旧版 spec 归档

## 优先级变更流程

1. **升级请求**: 任何团队成员可提出优先级升级请求
2. **E01 审查**: 首席架构师评估升级理由与技术影响
3. **决策**: 3 人共识（E01 + 相关领域专家）
4. **记录**: 在任务跟踪系统中更新优先级

## 资源分配策略

- **P0**: 100% 团队资源，暂停所有非 P0 工作
- **P1**: 70% 团队资源，P2-P3 可并行但不可阻塞
- **P2**: 50% 团队资源，与 P3 并行
- **P3**: 30% 团队资源，空闲时段执行
- **P4**: 10% 团队资源，季度末集中处理

## 技术债务管理

- 每个迭代周期（2 周）的技术债务偿还比例不低于 20%
- 技术债务项标记为 P4，按季度集中偿还
- 技术债务超过 20 项时触发 P1 升级
