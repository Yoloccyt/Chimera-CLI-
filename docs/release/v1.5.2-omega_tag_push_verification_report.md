### §10.5 P1 短板验证详情

**验证指标**: Windows GNU Build job 中 "Build binary" step 成功完成,无 `linker not found` 错误。

**v1.5.1-omega 基线**(run 29156147195):
- MinGW 安装步骤: ✅ 成功
- Verify MinGW toolchain: ✅ 成功(`which gcc` 输出 `/c/mingw64/bin/gcc`)
- Build binary: ❌ 失败(`error: linker C:/msys64/mingw64/bin/gcc.exe not found`)

**v1.5.2-omega 修复后**(run 29193406939):
- MinGW 安装步骤: ✅ 成功
- Configure MinGW linker: ✅ 成功(动态探测 `which gcc` + `cygpath -w`)
- Build binary: ✅ **成功**
- Windows GNU binary artifact: ✅ 已生成 `chimera-windows-x86_64.exe`

**结论**: §10.5 P1 短板已修复。根因是 msys2/setup-msys2@v2 在 `windows-2025-vs2026` runner 上将 MinGW 安装到 `C:/mingw64`,而 release.yml 硬编码 `C:/msys64/mingw64/bin/gcc.exe`。动态路径探测解决了路径不匹配问题。
