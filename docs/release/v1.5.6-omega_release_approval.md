## 2. Flaky Test 调查结论

### QA-DEFECT-001: `repo-wiki/store::tests::test_read_during_write_not_blocked`

- **失败现象**: SQLite WAL 并发读超时(`Elapsed(())`)
- **根因**: 35 crate 并行测试资源竞争(`spawn_blocking` 调度被拖累)
- **隔离复测**: debug 0.73s PASS / release 0.75s PASS
- **代码审计**: WAL 用 `pragma_update`、`spawn_blocking` 包装、`MutexGuard` 不跨 `.await`,全部符合项目铁律
- **结论**: **Resolved - Flaky Test**,不阻塞发布

### QA-DEFECT-002: `hcw-window/bench_retain_by_file_ids_hashset`

- **失败现象**: P50 延迟 6.577ms > 5ms 阈值
- **根因**: 并行竞争(主因)+ debug 模式开销(辅因)
- **隔离复测**: debug P50=4.934ms PASS / release P50=3.184ms PASS(余量 36%)
- **代码审计**: HashSet O(m) + `Vec::retain` O(n),算法复杂度达标
- **结论**: **Resolved - Flaky Test**,不阻塞发布
