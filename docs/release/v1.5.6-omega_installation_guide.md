## a. 软件介绍

### a.1 软件名称

**Chimera CLI**(代号 **NEXUS-OMEGA**,全称 Omni-Model Engineering Generative Architecture)—— 一个面向未来的多模型 AI 工程生成架构命令行工具。

### a.2 官方介绍与设计理念

Chimera CLI 是基于 Rust 2021 edition + Tokio 异步运行时构建的 35 crate workspace 项目,严格遵循 10 层架构(L1-L10)和 **OMEGA 四定律**:

| 定律 | 全称 | 工程实现 | 对应 crate |
|------|------|---------|-----------|
| **Ω-Sparse** | 全维稀疏(工具/上下文/记忆/审计/预算) | 五维度掩码稀疏化 | `osa-coordinator` |
| **Ω-Compress** | 层次化上下文压缩 | 四级窗口(4K/32K/128K/1M)+ 神经形态记忆 | `hcw-window` + `mlc-engine` |
| **Ω-Evolve** | 自组织在线进化 | GRPO 风格进化 + DPO 偏好优化 | `gsoe-evolution` + `auto-dpo` |
| **Ω-Event** | 事件驱动架构 | Tokio broadcast 跨层解耦 | `event-bus` |

项目的设计基因来源于 **Claude Code 尸检教训**(避免孤儿调用、裸奔命令、内存爆炸等系统性问题)、**Hermes 基因**、**Qoder 骨骼**,并融合 **DeepSeek V4 / Kimi K2.7 / GLM 5.2 / Minimax M3 / Qwen 3.7 Plus** 五大模型灵魂,共计 **37 个创新点**(22 个第一代 + 15 个第三代)。

核心目标是为多模型 AI 应用提供一个**安全、可观测、可进化**的异步执行框架。所有 35 个 crate 均声明 `#![forbid(unsafe_code)]`(编译期保证零 unsafe),配合 SecCore 零信任沙箱、Merkle 审计链、OWASP Top 10 渗透测试(20/20 全部拦截),从根本上杜绝传统 AI 代码助手的安全隐患。

性能层面,三层路由 p95 延迟 78.79µs(目标 ≤ 2ms,25× 余量),SSRA 100 模板融合 5.64µs,WAL 崩溃恢复 1000 次零数据丢失,适用于对延迟与可靠性敏感的生产环境。

### a.3 核心功能模块

Chimera CLI 通过 `chimera` 命令对外暴露以下子命令(完整命令树):

```text
chimera
  ├── run <prompt>          # 运行单次任务(不进入 Quest 长期任务流程)
  ├── tui                    # 启动 TUI 交互界面(基于 ratatui)
  ├── quest <action>         # Quest 长期任务管理
  │     ├── list             # 列出所有 Quest(含进行中/已完成/已取消)
  │     ├── show <id>        # 查看 Quest 详情
  │     ├── cancel <id>      # 取消 Quest(触发检查点保存)
  │     └── checkpoint <id>  # 为 Quest 创建检查点(LHQP 长期持久化)
  ├── config <action>        # 配置管理
  │     ├── init             # 生成默认 omega.yaml
  │     ├── list             # 列出当前生效配置项
  │     ├── show             # 显示完整配置(JSON 格式)
  │     └── path             # 显示配置文件路径
  ├── wiki <query>           # Wiki 语义检索(repo-wiki crate)
  └── parliament <proposal>  # 议会审议(多模型议会表决)
```

**全局参数**:

| 参数 | 说明 |
|------|------|
| `--config <FILE>` | 指定配置文件路径(默认 `~/.aether/omega.yaml`) |
| `-v` / `--verbose` | 启用详细日志(等价于 `RUST_LOG=debug`) |
| `--version` | 输出版本号 |
| `--help` | 输出帮助 |

> **品牌一致性说明**: binary 内部产物名为 `aether`(由 `crates/chimera-cli/Cargo.toml` 的 `[[bin]] name = "aether"` 决定),CI/Docker 中重命名为 `chimera` 以保持对外品牌一致。`--version` 输出格式正则 `^(aether|chimera) \d+\.\d+\.\d+` 与 CI 完全一致。

### a.4 适用场景

- **长上下文 AI Agent 任务**:HCW 四级窗口(4K/32K/128K/1M)支持超长上下文,经 OSA 稀疏化后实际加载量大幅压缩
- **多模型协调与路由**:CACR 成本感知模型路由,根据任务复杂度自动选择主/备模型
- **知识沉淀与进化学习**:RepoWiki 代码 Wiki + GSOE 在线进化 + Auto-DPO 偏好优化
- **安全沙箱执行**:SecCore 零信任沙箱 + ASA 对抗审计 + QEEP 孤儿调用检测
- **多平台 IDE 集成**:CHTC 跨平台桥接 5 大 IDE,MCP Mesh 跨进程通信

### a.5 与同类工具的优势对比

| 维度 | Chimera CLI | Claude Code | Cursor | Aider |

**核心优势**: Chimera CLI 通过 OMEGA 四定律从根本上解决了 Claude Code 尸检发现的"孤儿调用(5.4%)""裸奔命令""1M Token 暴力加载"等系统性问题,是首个将"安全 + 可观测 + 可进化"作为一等公民的 AI Agent CLI。

## b. 操作系统环境检测与包管理器选择

### b.1 操作系统自动检测命令

在安装前,请先确认您的操作系统与架构。以下命令在各平台原生 shell 中执行:

**Linux / macOS**(bash/zsh):

```bash
# 检测操作系统
uname -s
# 预期输出: Linux 或 Darwin

# 检测处理器架构
uname -m
# 预期输出: x86_64 / aarch64 / arm64
```

**Windows**(PowerShell 5.1+):

```powershell
# 检测操作系统版本
[System.Environment]::OSVersion.VersionString
# 预期输出: Microsoft Windows NT 10.0.22631.0 (类似)

# 检测处理器架构
$env:PROCESSOR_ARCHITECTURE
# 预期输出: AMD64 (即 x86_64) 或 ARM64
```

**macOS**(额外提供):

```bash
sw_vers
# 预期输出:
# ProductName:    macOS
# ProductVersion: 14.5
# BuildVersion:   23F79
```

### b.3 自动检测脚本

**Linux / macOS 自动检测**:

```bash
#!/usr/bin/env bash
# 检测 OS / ARCH 并输出推荐安装方式
OS="$(uname -s)"
ARCH="$(uname -m)"

case "${OS}" in
    Linux*)  PLATFORM="linux";;
    Darwin*) PLATFORM="macos";;
    *)       echo "不支持的 OS: ${OS}"; exit 1;;
esac

case "${ARCH}" in
    x86_64|amd64)  ARCH_NORM="x86_64";;
    aarch64|arm64) ARCH_NORM="aarch64";;
    *)             echo "不支持的架构: ${ARCH}"; exit 1;;
esac

echo "检测到平台: ${PLATFORM}/${ARCH_NORM}"
echo "推荐方式: curl -fsSL https://raw.githubusercontent.com/<user>/chimera-cli/main/install.sh | sh"

# 检测包管理器(可选)
if command -v brew >/dev/null 2>&1; then
    echo "已安装: Homebrew (可选: brew install chimera-cli)"
elif command -v apt >/dev/null 2>&1; then
    echo "已安装: apt (可选: 下载 .deb 后 sudo apt install ./chimera-cli_*.deb)"
elif command -v docker >/dev/null 2>&1; then
    echo "已安装: Docker (可选: docker pull ghcr.io/<user>/chimera-cli:v1.5.6-omega)"
fi
```

**Windows 自动检测**(PowerShell):

```powershell
# 检测架构
$arch = if ($env:PROCESSOR_ARCHITECTURE -match 'AMD64|X64') { 'x86_64' }
        elseif ($env:PROCESSOR_ARCHITECTURE -eq 'ARM64') { 'aarch64' }
        else { $env:PROCESSOR_ARCHITECTURE }
Write-Host "检测到平台: windows/$arch"
Write-Host "推荐方式: iex (irm https://raw.githubusercontent.com/<user>/chimera-cli/main/install.ps1)"

# 检测包管理器(可选)
if (Get-Command scoop -ErrorAction SilentlyContinue) {
    Write-Host "已安装: Scoop (可选: scoop install chimera-cli)"
}
if (Get-Command choco -ErrorAction SilentlyContinue) {
    Write-Host "已安装: Chocolatey (可选: choco install chimera-cli)"
}
if (Get-Command docker -ErrorAction SilentlyContinue) {
    Write-Host "已安装: Docker (可选: docker pull ghcr.io/<user>/chimera-cli:v1.5.6-omega)"
}
```

## c. 具体安装命令

### c.1 Windows 安装

#### 方式 1: PowerShell 一键脚本(推荐)

`install.ps1` 会自动检测架构、从 GitHub Release 下载对应 binary、可选 SHA256 校验、安装到 `%LOCALAPPDATA%\Programs\chimera\`、注入用户 PATH 并验证 `chimera --version`。

**公有仓库**(可省略鉴权 header):

```powershell
# 直接通过 irm 拉取并 iex 执行
iex (irm https://raw.githubusercontent.com/<user>/chimera-cli/main/install.ps1)
```

**私有仓库**(必须携带 `GITHUB_TOKEN`):

```powershell
# 1. 设置 GITHUB_TOKEN 环境变量(替换为您的 token)
$env:GITHUB_TOKEN = 'ghp_xxx'

# 2. 构造鉴权 header(raw.githubusercontent.com 对私有仓库 raw 内容拒绝匿名访问)
$headers = @{ Authorization = "Bearer $env:GITHUB_TOKEN" }

# 3. 拉取并执行安装脚本
iex (irm https://raw.githubusercontent.com/<user>/chimera-cli/main/install.ps1 -Headers $headers)
```

**可选参数**:

```powershell
# 指定版本(默认拉取 latest)
.\install.ps1 -Version v1.5.6-omega

# 指定安装目录(默认 $env:LOCALAPPDATA\Programs\chimera)
.\install.ps1 -InstallDir C:\Tools\chimera

# 跳过 SHA256 校验(不推荐生产环境)
.\install.ps1 -SkipVerify

# 仅配置 Rust 工具链环境变量(源码构建场景,不下载 binary)
.\install.ps1 -SetupEnv
```

**管理员权限说明**: 默认安装到用户目录 `%LOCALAPPDATA%\Programs\chimera\`,**无需管理员权限**。若指定 `-InstallDir C:\Program Files\...` 等系统目录,需以管理员身份运行 PowerShell。

#### 方式 2: Scoop

> 仅当社区已发布 scoop bucket 时可用。若未发布,请回退到方式 1。

```powershell
# 1. 添加 scoop bucket(替换为实际仓库)
scoop bucket add chimera-cli https://github.com/<user>/scoop-chimera-cli.git

# 2. 安装
scoop install chimera-cli

# 3. 更新
scoop update chimera-cli
```

#### 方式 3: Chocolatey

> 仅当社区已发布 choco 包时可用。若未发布,请回退到方式 1。

```powershell
# 安装(需管理员权限)
choco install chimera-cli -y

# 更新
choco upgrade chimera-cli -y
```

#### 方式 4: 直接下载二进制

从 [GitHub Releases](https://github.com/<user>/chimera-cli/releases) 下载对应平台资产:

- **Windows x86_64**: `chimera-windows-x86_64.exe`

```powershell
# 1. 下载 binary(私有仓库需加 -Headers @{ Authorization = "Bearer $env:GITHUB_TOKEN" })
$env:GITHUB_TOKEN = 'ghp_xxx'  # 私有仓库必填
$headers = @{ Authorization = "Bearer $env:GITHUB_TOKEN"; "User-Agent" = "chimera-install" }
Invoke-WebRequest `
    -Uri "https://github.com/<user>/chimera-cli/releases/download/v1.5.6-omega/chimera-windows-x86_64.exe" `
    -OutFile "chimera-windows-x86_64.exe" `
    -Headers $headers

# 2. 创建安装目录(用户目录,无需管理员)
$installDir = "$env:LOCALAPPDATA\Programs\chimera"
New-Item -ItemType Directory -Force -Path $installDir | Out-Null

# 3. 移动 binary(重命名为 chimera.exe)
Move-Item chimera-windows-x86_64.exe "$installDir\chimera.exe" -Force

# 4. 注入用户 PATH(幂等)
$userPath = [Environment]::GetEnvironmentVariable('PATH', 'User')
if ($userPath -notlike "*$installDir*") {
    $newPath = "$installDir;$userPath"
    [Environment]::SetEnvironmentVariable('PATH', $newPath, 'User')
    Write-Host "PATH 已更新,请重新打开 PowerShell 使其生效" -ForegroundColor Yellow
}

# 5. 验证(需新开终端或刷新当前会话 PATH)
$env:Path = "$installDir;$env:Path"
chimera --version
# 预期输出: chimera 1.5.6-omega
```

#### 方式 2: Homebrew

> 仅当社区已发布 brew tap 时可用。若未发布,请回退到方式 1。


#### 方式 3: 直接下载二进制

根据 CPU 架构选择对应资产:

- **Intel Mac**: `chimera-macos-x86_64`
- **Apple Silicon (M1/M2/M3/M4)**: `chimera-macos-aarch64`

```bash
# 1. 检测架构(Apple Silicon 输出 arm64,Intel 输出 x86_64)
ARCH=$(uname -m)
echo "架构: $ARCH"

# 2. 下载(私有仓库需加 -H "Authorization: Bearer $GITHUB_TOKEN")
export GITHUB_TOKEN='ghp_xxx'  # 私有仓库必填
if [ "$ARCH" = "arm64" ]; then
    ARTIFACT="chimera-macos-aarch64"
else
    ARTIFACT="chimera-macos-x86_64"
fi
curl -fSL -H "Authorization: Bearer $GITHUB_TOKEN" \
  -o "$ARTIFACT" \
  "https://github.com/<user>/chimera-cli/releases/download/v1.5.6-omega/$ARTIFACT"

# 3. 安装到 ~/.local/bin(无需 sudo)
chmod +x "$ARTIFACT"
mkdir -p ~/.local/bin
mv "$ARTIFACT" ~/.local/bin/chimera

# 4. 确保 ~/.local/bin 在 PATH 中
case "$(basename "$SHELL")" in
    zsh)  RC_FILE="$HOME/.zshrc";;
    bash) RC_FILE="$HOME/.bashrc";;
    fish) RC_FILE="$HOME/.config/fish/config.fish";;
    *)    RC_FILE="$HOME/.profile";;
esac
grep -q 'export PATH="$HOME/.local/bin:$PATH"' "$RC_FILE" 2>/dev/null || \
    echo 'export PATH="$HOME/.local/bin:$PATH"' >> "$RC_FILE"
source "$RC_FILE"

# 5. 验证
chimera --version
# 预期输出: chimera 1.5.6-omega
```

#### 方式 4: 系统目录安装(需 sudo)

```bash
sudo install -m 0755 chimera-macos-aarch64 /usr/local/bin/chimera
chimera --version
```

### c.3 Linux 安装

#### 方式 1: 一键脚本(推荐)

```bash
# 公有仓库
curl -fsSL https://raw.githubusercontent.com/<user>/chimera-cli/main/install.sh | sh

# 私有仓库(必须携带 GITHUB_TOKEN)
export GITHUB_TOKEN='ghp_xxx'
curl -fsSL -H "Authorization: Bearer $GITHUB_TOKEN" \
  https://raw.githubusercontent.com/<user>/chimera-cli/main/install.sh | sh
```

**可选参数**:

```bash
# 指定版本
sh install.sh --version v1.5.6-omega

# 指定安装目录(默认 ~/.local/bin;系统目录需 sudo)
sudo sh install.sh --install-dir /usr/local/bin

# 跳过 SHA256 校验(不推荐)
sh install.sh --skip-verify

# 查看帮助
sh install.sh --help
```

`install.sh` 内部逻辑:
1. 检测 `uname -s` / `uname -m` → 确定 `linux/x86_64` 等平台元组
2. 通过 GitHub API 获取 latest release tag(或使用 `--version` 指定)
3. 下载 `chimera-linux-<arch>` binary + 可选 `checksums.txt`
4. SHA256 校验(`sha256sum` 或 macOS 的 `shasum -a 256`)
5. 安装到 `--install-dir`(默认 `~/.local/bin`,系统目录自动 `sudo`)
6. 按当前 `$SHELL` 写入对应 rc 文件(幂等,带 `# chimera-cli install` marker)
7. 验证 `chimera --version`,正则 `^(aether|chimera) [0-9]+\.[0-9]+\.[0-9]+`

> 仅当 Release 附带 `.deb` 资产时可用。当前 Release 主要提供单文件 binary,可回退到方式 1。

```bash

case "$(basename "$SHELL")" in
    zsh)  RC_FILE="$HOME/.zshrc";;
    bash) RC_FILE="$HOME/.bashrc";;

esac
grep -q 'export PATH="$HOME/.local/bin:$PATH"' "$RC_FILE" 2>/dev/null || \
    printf '\n# chimera-cli install\nexport PATH="%s:$PATH"\n' "$HOME/.local/bin" >> "$RC_FILE"
```

## e. 常见问题排查

### e.1 网络问题

**症状**: 安装脚本下载二进制超时或失败

**识别**:

```text
curl: (7) Failed to connect to github.com port 443: Connection timed out
curl: (22) The requested URL returned error: 404 Not Found
```

**解决**:

1. **私有仓库鉴权**: 确认已设置 `GITHUB_TOKEN` 并通过 `-H "Authorization: Bearer $GITHUB_TOKEN"` 显式传递 header。仅设置环境变量不会自动被 `curl`/`irm` 加入 header。

   ```bash
   # Linux / macOS
   export GITHUB_TOKEN='ghp_xxx'
   curl -fsSL -H "Authorization: Bearer $GITHUB_TOKEN" \
     https://raw.githubusercontent.com/<user>/chimera-cli/main/install.sh | sh
   ```

   ```powershell
   # Windows
   $env:GITHUB_TOKEN = 'ghp_xxx'
   $headers = @{ Authorization = "Bearer $env:GITHUB_TOKEN" }
   iex (irm https://raw.githubusercontent.com/<user>/chimera-cli/main/install.ps1 -Headers $headers)
   ```

2. **配置 HTTP 代理**(适用于企业防火墙):

   ```bash
   export HTTPS_PROXY=http://proxy.company.com:8080
   export HTTP_PROXY=http://proxy.company.com:8080
   ```

3. **DNS 污染**(常见于中国大陆): `raw.githubusercontent.com` 可能被 DNS 污染。解决方法:
   - 配置可信 DNS(如 `1.1.1.1` / `8.8.8.8`)
   - 使用 [GitHub Releases](https://github.com/<user>/chimera-cli/releases) 页面手动下载(浏览器可能通过不同路径访问)
   - 克隆仓库后本地执行 `sh install.sh` / `.\install.ps1`

4. **降级为 Docker 部署**(若可拉取 GHCR 镜像):

   ```bash
   docker pull ghcr.io/<user>/chimera-cli:v1.5.6-omega
   ```

5. **从源码构建**(需 Rust 工具链,见 c.5 节)

### e.2 依赖冲突 / glibc 版本不匹配

**症状**: Linux 安装后运行报错

**识别**:

```text
./chimera: /lib64/libc.so.6: version 'GLIBC_2.28' not found (required by ./chimera)
```

2. **从源码构建**(使用本机 glibc 链接):

   ```bash
   git clone https://github.com/<user>/chimera-cli.git
   cd chimera-cli
   cargo build --workspace --release
   sudo install -m 0755 target/release/aether /usr/local/bin/chimera
   ```

3. **升级 glibc**(风险高,不推荐,可能破坏系统)

**解决**:

### e.10 Windows GNU 环境下 fuzz 编译失败

**症状**:

```text

```

**根因**: libFuzzer 的 Windows 适配依赖 MSVC(`__declspec(dllimport)`),本项目使用 MinGW/GNU 工具链无法编译 `libfuzzer-sys`。

**解决**:

- **本地跳过 fuzz 编译**: 该命令不是发布阻塞项;fuzz 实际执行委托 Linux CI(`.github/workflows/fuzz.yml`)
- **本地静态验证配置完整性**:

  ```powershell
  .\scripts\check_fuzz_config.ps1
  ```

- **在 Linux/macOS + nightly 环境下本地执行 fuzz**:

  ```bash
  cargo +nightly fuzz run <target>
  ```

## 附录: 环境变量参考

### 运行时环境变量

| 变量名 | 作用 | 默认值 | 说明 |
|--------|------|--------|------|
| `AETHER_CONFIG` | 自定义配置文件路径 | `~/.aether/omega.yaml` | 等价于 `--config` 参数 |
| `AETHER_MODEL__PRIMARY` | 主模型 ID | (配置文件) | 嵌套字段用 `__` 分隔 |
| `AETHER_MODEL__FALLBACK` | 备用模型 ID | (配置文件) | 主模型失败时切换 |
| `RUST_LOG` | 日志级别 | `info` | `debug` / `info` / `warn` / `error`;`--verbose` 等价于 `debug` |
| `RUST_BACKTRACE` | panic 反向追踪 | `1`(Docker 镜像内) | 排查问题时建议开启 |
| `GITHUB_TOKEN` | GitHub API 鉴权 | (未设置) | 私有仓库安装/下载必需 |

### 开发/构建环境变量(Windows)

| 变量名 | 作用 | 推荐值 |
|--------|------|--------|
| `CARGO_HOME` | Cargo 主目录 | `D:\Chimera CLI\.toolchain\cargo` |
| `RUSTUP_HOME` | Rustup 主目录 | `D:\Chimera CLI\.toolchain\rustup` |
| `TMP` / `TEMP` | 临时目录 | `D:\Chimera CLI\tmp` |
| `PATH` | 可执行文件搜索路径 | 含 `D:\Chimera CLI\.toolchain\cargo\bin` 与 `D:\msys64\mingw64\bin` |
| `RUST_MIN_STACK` | Rust 最小栈大小 | `33554432`(32MB,clippy OOM 缓解) |
| `CARGO_INCREMENTAL` | 增量编译 | `0`(GNU 工具链 ICE 缓解) |
| `CARGO_TARGET_DIR` | 构建产物目录 | (默认 `target/`;D 盘空间不足时可重定向至 `E:\chimera-target`) |
| `SQLITE_ENABLE_FTS5` | 启用 SQLite FTS5 全文索引 | `1`(repo-wiki crate 需要) |

### 配置加载优先级(由低到高)

1. **内置默认值**: `ChimeraConfig::default()`(编译期常量)
2. **配置文件**: `~/.aether/omega.yaml`(YAML 格式,可被 `--config` 覆盖)
3. **环境变量**: 前缀 `AETHER_`,嵌套字段用 `__`(如 `AETHER_MODEL__PRIMARY=premium`)
4. **CLI 参数**: `--config`、`--verbose` 等(优先级最高)

**多源加载**使用 Figment 框架,后者覆盖前者。
