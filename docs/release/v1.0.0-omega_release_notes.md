- binary 体积:`strip = true` / `lto = true` / `opt-level = "z"` / `panic = "abort"` / `codegen-units = 1`

## 4. 安全特性

### 4.1 `#![forbid(unsafe_code)]` 40/40 全覆盖

所有 34 个 workspace crate + 6 个附属 crate(fuzz / benches 等)均声明 `#![forbid(unsafe_code)]`,这是项目最硬的安全红线。Week 8 验证 40/40 覆盖,无任何 crate 放松约束。

**关键决策**:rusqlite `bundled` feature 与 `forbid(unsafe_code)` 兼容——`forbid` 是 crate 级,只扫描当前 crate 源码,不传播到依赖,因此 SQLite 持久化存储可用于 mlc-engine / quest-engine。

### 4.2 OWASP Top 10 渗透测试(20/20 通过)

`tests/security/owasp_top10.rs` 覆盖 A01-A10,每项含正常用例 + 攻击用例:

- SecCore 零信任沙箱拒绝未知命令(A04 Insecure Design)
- 注入字符(`;` `|` `$()` 等)被 SecCore 过滤(A03 Injection)
- 跨层通信仅走 EventBus,无向上依赖(A08 Software & Data Integrity)
- AHIRT 反黑客红队 5 分钟周期审计(A09 Security Logging)

### 4.3 模糊测试(cargo-fuzz 3 target)

`fuzz/` 独立 crate(不污染主 workspace),3 target:

- `quest_parse`:Quest 解析器模糊
- `seccore_sandbox`:SecCore 沙箱模糊
- `event_serialize`:事件序列化模糊

> **限制**:`cargo fuzz` 要求 nightly toolchain,3 target 已就绪待 CI 环境 nightly 完整运行。

### 4.4 依赖审计(cargo-audit 无高危)

手动检查 Cargo.lock 中 13 个关键依赖版本,对照 RustSec Advisory Database,无 High/Critical 漏洞。

> **限制**:cargo-audit 安装因网络超时未完成,fallback 采用手动审计。后续 CI 应集成 `cargo audit` 自动化。
