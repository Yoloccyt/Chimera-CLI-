## 1. 路线图概览

### 1.1 v1.1.0 核心目标

v1.1.0-omega 聚焦 **6 项从 v1.0.0 延后的核心功能** + **3 项独立占位实现替换**,从"功能完备但部分占位"升级为"生产级全实现"。本版本将突破 RC 阶段约束,允许跨层重构、引入新 crate、新依赖,但严格遵循 10 层架构与依赖方向规则。

### 1.2 功能清单(9 项)

| 编号 | 功能 | 类型 | 当前状态 | 目标状态 | 优先级 |
|------|------|------|---------|---------|--------|
| F1 | chimera-cli L10 真实编排接线 | 延后功能 | Stage 0 占位 | 完整 L10 编排 | Must |
| F2 | nexus-core rusqlite 下沉到 L3 | 架构重构 | L1 直连 rusqlite | L3 封装存储层 | Must |
| F3 | seccore 内核级沙箱(ADR-001) | 安全增强 | 降级版本 | gVisor/WCIF 实现 | Should |
| F4 | GSOE 真实强化学习策略 | 智能进化 | GRPO 规则占位 | PPO/GRPO 真实模型 | Should |
| F5 | NMC 多模态感知器 ONNX | 多模态 | EncodingFailed 占位 | ort ONNX 接入 | Should |
| F6 | MCP Mesh 2PC 真实跨进程 | 分布式 | mock 单进程 | 真实 2PC + 跨进程 | Must |
| F7 | MTPE 真实多步预测 | 占位替换 | pseudo_predictions | 接入模型推理 | Could |
| F8 | RepoWiki 语义嵌入 | 占位替换 | SHA-256 哈希扩展 | NMC CLV 512-dim | Could |
| F9 | SCC 持久化 WAL | 占位替换 | InMemoryWal | SqliteWal | Could |

### 1.3 MoSCoW 优先级

- **Must**:F1(L10 编排)、F2(rusqlite 下沉)、F6(MCP Mesh 2PC)— 阻塞 v1.1.0 发布
- **Should**:F3(内核沙箱)、F4(GSOE RL)、F5(NMC ONNX)— 强烈建议,影响安全/智能/多模态核心能力
- **Could**:F7(MTPE)、F8(RepoWiki)、F9(SCC WAL)— 周期内补齐,不影响发布
