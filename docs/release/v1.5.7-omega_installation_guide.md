## a. 软件介绍

### a.1 软件名称

**chimela CLI**(代号 **NEXUS-OMEGA**,全称 Omni-Model Engineering Generative Architecture)—— 一个面向未来的多模型 AI 工程生成架构命令行工具。

### a.2 官方介绍与设计理念

chimela CLI 是基于 Rust 2021 edition + Tokio 异步运行时构建的 35 crate workspace 项目,严格遵循 10 层架构(L1-L10)和 **OMEGA 四定律**:

| 定律 | 全称 | 工程实现 | 对应 crate |
|------|------|---------|-----------|
| **Ω-Sparse** | 全维稀疏(工具/上下文/记忆/审计/预算) | 五维度掩码稀疏化 | `osa-coordinator` |
| **Ω-Compress** | 层次化上下文压缩 | 四级窗口(4K/32K/128K/1M)+ 神经形态记忆 | `hcw-window` + `mlc-engine` |
| **Ω-Evolve** | 自组织在线进化 | GRPO 风格进化 + DPO 偏好优化 | `gsoe-evolution` + `auto-dpo` |
| **Ω-Event** | 事件驱动架构 | Tokio broadcast 跨层解耦 | `event-bus` |

项目的设计目标是为多模型 AI 应用提供一个**安全、可观测、可进化**的异步执行框架。所有 35 个 crate 均声明 `#![forbid(unsafe_code)]`(编译期保证零 unsafe),配合 SecCore 零信任沙箱、Merkle 审计链、OWASP Top 10 渗透测试(20/20 全部拦截),从根本上杜绝传统 AI 代码助手的安全隐患。

性能层面,三层路由 p95 延迟 78.79µs(目标 ≤ 2ms,25× 余量),SSRA 100 模板融合 5.64µs,WAL 崩溃恢复 1000 次零数据丢失,适用于对延迟与可靠性敏感的生产环境。

### a.3 核心功能模块

chimela CLI 通过 `chimela` 命令对外暴露以下子命令(完整命令树):

```text
chimela
  ├── run <prompt>          # 运行单次任务(不进入 Quest 长期任务流程)
  ├── tui                    # 启动 TUI 交互界面(基于 ratatui)
  ├── quest <action>         # Quest 长期任务管理
  │     ├── list             # 列出所有 Quest(含进行中/已完成/已取消)
  │     ├── show <id>        # 查看 Quest 详情
  │     ├── cancel <id>      # 取消 Quest(触发检查点保存)
  │     └── checkpoint <id>  # 为 Quest 创建检查点(LHQP 长期持久化)
  ├── config <action>        # 配置管理
  │     ├── init             # 生成默认 omega.yaml
  │     ├── list             # 列出当前生效配置项
  │     ├── show             # 显示完整配置(JSON 格式)
  │     └── path             # 显示配置文件路径
  ├── wiki <query>           # Wiki 语义检索(repo-wiki crate)
  └── parliament <proposal>  # 议会审议(多模型议会表决)
```

**全局参数**:

| 参数 | 说明 |
|------|------|
| `--config <FILE>` | 指定配置文件路径(默认 `~/.aether/omega.yaml`) |
| `-v` / `--verbose` | 启用详细日志(等价于 `RUST_LOG=debug`) |
| `--version` | 输出版本号 |
| `--help` | 输出帮助 |

> **品牌一致性说明**: binary 内部 cargo 产物名为 `aether`(由 `crates/chimera-cli/Cargo.toml` 的 `[[bin]] name = "aether"` 决定),CI/Docker 中重命名为 `chimela` 作为用户侧入口。`chimera` 与 `aether` 均作为兼容别名保留。`--version` 输出格式正则 `^(aether|chimera|chimela) \d+\.\d+\.\d+` 与 CI 完全一致。

### a.4 适用场景

- **长上下文 AI Agent 任务**: HCW 四级窗口(4K/32K/128K/1M)支持超长上下文,经 OSA 稀疏化后实际加载量大幅压缩。
- **多模型协调与路由**: CACR 成本感知模型路由,根据任务复杂度自动选择主/备模型。
- **知识沉淀与进化学习**: RepoWiki 代码 Wiki + GSOE 在线进化 + Auto-DPO 偏好优化。
- **安全沙箱执行**: SecCore 零信任沙箱 + ASA 对抗审计 + QEEP 孤儿调用检测。
- **多平台 IDE 集成**: CHTC 跨平台桥接 5 大 IDE,MCP Mesh 跨进程通信。

### a.5 与同类工具的优势对比

| 维度 | chimela CLI | Claude Code | Cursor | Aider |
|------|-------------|-------------|--------|-------|
| **架构** | 10 层 + OMEGA 四定律 | 单体 | IDE 插件 | 单体脚本 |
| **unsafe 保证** | 35/35 crate `#![forbid(unsafe_code)]` | N/A | N/A | N/A |
| **跨层通信** | Event Bus + MCP Mesh(解耦) | 紧耦合 | IDE 内嵌 | 单进程 |
| **稀疏化** | 五维度掩码(工具/上下文/记忆/审计/预算) | 无 | 无 | 无 |
| **进化能力** | GRPO + DPO 在线学习 | 无 | 无 | 无 |
| **审计** | Merkle 链 + AHIRT 红队 | 日志 | 无 | 无 |
| **部署形态** | CLI + TUI + Docker + MCP | CLI | GUI | CLI |

**核心优势**: chimela CLI 通过 OMEGA 四定律解决了传统 AI 代码助手的系统性风险(孤儿调用、裸奔命令、大上下文暴力加载),是首个将"安全 + 可观测 + 可进化"作为一等公民的 AI Agent CLI。

## b. 操作系统环境检测与包管理器选择

### b.1 操作系统自动检测命令

在安装前,请先确认您的操作系统与架构。以下命令在各平台原生 shell 中执行:

**Linux / macOS**(bash/zsh):

```bash
# 检测操作系统;预期输出: Linux 或 Darwin
uname -s

# 检测处理器架构;预期输出: x86_64 / aarch64 / arm64
uname -m
```

**Windows**(PowerShell 5.1+):

```powershell
# 检测操作系统版本;预期输出类似: Microsoft Windows NT 10.0.22631.0
[System.Environment]::OSVersion.VersionString

# 检测处理器架构;预期输出: AMD64(即 x86_64)或 ARM64
$env:PROCESSOR_ARCHITECTURE
```

**macOS**(额外提供):

```bash
# 显示 macOS 版本与构建号
sw_vers
```

### b.3 自动检测脚本

**Linux / macOS 自动检测**:

```bash
#!/usr/bin/env bash
# 检测 OS / ARCH 并输出推荐安装方式
OS="$(uname -s)"
ARCH="$(uname -m)"

case "${OS}" in
    Linux*)  PLATFORM="linux";;
    Darwin*) PLATFORM="macos";;
    *)       echo "不支持的 OS: ${OS}"; exit 1;;
esac

case "${ARCH}" in
    x86_64|amd64)  ARCH_NORM="x86_64";;
    aarch64|arm64) ARCH_NORM="aarch64";;
    *)             echo "不支持的架构: ${ARCH}"; exit 1;;
esac

echo "检测到平台: ${PLATFORM}/${ARCH_NORM}"
echo "推荐方式: curl -fsSL https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.sh | sh"

# 检测包管理器(可选提示)
if command -v brew >/dev/null 2>&1; then
    echo "已安装 Homebrew (官方 tap 待初始化,暂请使用 install.sh)"
elif command -v apt >/dev/null 2>&1; then
    echo "已安装 apt (可下载 .deb 后 sudo apt install ./chimela-cli_1.5.7-omega_amd64.deb)"
elif command -v docker >/dev/null 2>&1; then
    echo "已安装 Docker (可选: docker pull ghcr.io/yoloccyt/chimera-cli:v1.5.7-omega)"
fi
```

**Windows 自动检测**(PowerShell):

```powershell

Write-Host "检测到平台: windows/$arch"

    Write-Host "已安装 Chocolatey (官方包待初始化,暂请使用 install.ps1)"
```

## c. 具体安装命令

### c.1 Windows 安装

#### 方式 1: PowerShell 一键脚本(推荐)

**公有仓库**(省略鉴权 header):

```powershell
# 从 raw.githubusercontent.com 下载 install.ps1 并通过 iex 执行
# iex: Invoke-Expression, irm: Invoke-RestMethod
iex (irm https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.ps1)
```

**私有仓库**(必须携带 GITHUB_TOKEN):

```powershell
# 1) 将 GitHub personal access token 写入当前会话环境变量
$env:GITHUB_TOKEN='ghp_xxx'

# 2) 构造 Authorization header,IRM 不会自动读取 $env:GITHUB_TOKEN
$headers = @{ Authorization = "Bearer $env:GITHUB_TOKEN" }

# 3) 下载并执行安装脚本;脚本内部下载 Release binary 时也会读取 $env:GITHUB_TOKEN
iex (irm https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.ps1 -Headers $headers)
```

> **权限说明**: 默认安装到 `%LOCALAPPDATA%\Programs\chimela`(用户目录),**不需要管理员权限**。若指定 `-InstallDir 'C:\Windows'` 等系统目录,则需要以管理员身份运行 PowerShell。

常用可选参数:

```powershell
# 指定版本安装
.\install.ps1 -Version v1.5.7-omega

# 指定安装目录(需要管理员权限写入系统目录)
.\install.ps1 -InstallDir 'C:\Program Files\chimela'

# 使用 HTTP/HTTPS 代理(企业网络场景)
.\install.ps1 -Proxy 'http://proxy.company.com:8080'

# 跳过 SHA256 校验
.\install.ps1 -SkipVerify

# 仅设置 Rust 工具链环境变量,不下载 binary(源码构建场景)
.\install.ps1 -SetupEnv
```

#### 方式 2: 手动下载安装

```powershell
# 1) 设置版本号
$Version = 'v1.5.7-omega'

# 2) 下载预编译 binary(私有仓库需加 -Headers @{ Authorization = "Bearer $env:GITHUB_TOKEN" })
$Url = "https://github.com/Yoloccyt/Chimera-CLI-/releases/download/${Version}/chimela-windows-x86_64.exe"
$OutFile = "$env:TEMP\chimela-windows-x86_64.exe"
Invoke-WebRequest -Uri $Url -OutFile $OutFile -UseBasicParsing

# 3) 创建安装目录(用户级,无需管理员)
$InstallDir = "$env:LOCALAPPDATA\Programs\chimela"
New-Item -ItemType Directory -Force -Path $InstallDir | Out-Null

# 4) 移动 binary 到安装目录并命名为 chimela.exe
Move-Item -Path $OutFile -Destination "$InstallDir\chimela.exe" -Force

# 5) 将安装目录追加到用户 PATH(持久化)
[Environment]::SetEnvironmentVariable('Path', "$InstallDir;$([Environment]::GetEnvironmentVariable('Path', 'User'))", 'User')

# 6) 验证
chimela --version
```

#### 方式 3: Scoop / Chocolatey(模板渠道,待初始化)

```powershell
# Scoop bucket 尚未初始化,以下命令待官方 bucket 可用后执行
scoop bucket add chimela-cli https://github.com/Yoloccyt/scoop-chimela-cli.git
scoop install chimela-cli

# Chocolatey 包尚未初始化,以下命令待官方包可用后执行
choco install chimela-cli
```

在仓库初始化前,请使用方式 1 或方式 2。

### c.2 Linux / macOS 安装

#### 方式 1: curl 一键脚本(推荐)

**公有仓库**(省略鉴权 header):

```bash
# -f: HTTP 错误时失败; -s: 静默; -S: 错误时显示; -L: 跟随重定向
# 通过管道传给 sh 执行 install.sh
curl -fsSL https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.sh | sh
```

**私有仓库**(必须携带 GITHUB_TOKEN):

```bash
# 1) 将 GitHub personal access token 写入当前 shell 环境变量
export GITHUB_TOKEN='ghp_xxx'

# 2) 显式在 HTTP header 中传递 Authorization;curl 不会自动读取 $GITHUB_TOKEN
curl -fsSL -H "Authorization: Bearer $GITHUB_TOKEN" \
  https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.sh | sh
```

> **权限说明**: 默认安装到 `~/.local/bin`(用户目录),**不需要 sudo**。若指定 `--install-dir /usr/local/bin`,则需要 `sudo`。

常用可选参数:

```bash
# 指定版本
sh install.sh --version v1.5.7-omega

# 指定系统级安装目录(需要 sudo)
sudo sh install.sh --install-dir /usr/local/bin

# 跳过 SHA256 校验
sh install.sh --skip-verify
```

#### 方式 2: 手动下载安装

```bash
# 1) 设置版本与平台
VERSION='v1.5.7-omega'
OS='linux'        # Linux 用户设为 linux;macOS 用户设为 macos
ARCH='x86_64'     # x86_64 或 aarch64(Apple Silicon / ARM 服务器)

# 2) 下载预编译 binary
# 私有仓库需在 curl 中增加 -H "Authorization: Bearer $GITHUB_TOKEN"
curl -fsSL -o /tmp/chimela "https://github.com/Yoloccyt/Chimera-CLI-/releases/download/${VERSION}/chimela-${OS}-${ARCH}"

# 3) 添加可执行权限
chmod +x /tmp/chimela

# 4) 移动到用户级 bin 目录(无需 sudo)
mkdir -p ~/.local/bin
mv /tmp/chimela ~/.local/bin/chimela

# 5) 确保 ~/.local/bin 在 PATH 中
export PATH="$HOME/.local/bin:$PATH"

# 6) 验证
chimela --version
```

#### 方式 3: Debian / Ubuntu `.deb` 包

```bash
# 1) 下载 .deb 包(amd64 架构)
# 私有仓库需加 -H "Authorization: Bearer $GITHUB_TOKEN"
curl -fsSL -o /tmp/chimela-cli_1.5.7-omega_amd64.deb \
  https://github.com/Yoloccyt/Chimera-CLI-/releases/download/v1.5.7-omega/chimela-cli_1.5.7-omega_amd64.deb

# 2) 使用 apt 安装本地 .deb 包(自动处理依赖,需要 sudo)
sudo apt install /tmp/chimela-cli_1.5.7-omega_amd64.deb

# 或直接使用 dpkg 安装(需要手动处理依赖)
sudo dpkg -i /tmp/chimela-cli_1.5.7-omega_amd64.deb

# 3) 验证
chimela --version
```

> **包名说明**: `.deb` 包文件名为 `chimela-cli_1.5.7-omega_amd64.deb`,包名(Package 字段)为 `chimela-cli`。安装后 `/usr/bin/chimela` 为主入口,`/usr/bin/chimera` 为软链接别名。

#### 方式 4: Homebrew(模板渠道,待初始化)

```bash
# 官方 Homebrew tap 尚未初始化,以下命令待 tap 可用后执行
brew tap yoloccyt/chimela-cli
brew install chimela-cli
```

在仓库初始化前,请使用方式 1 或方式 2。

### c.3 Docker 安装

chimela CLI 提供基于 distroless 的多阶段 Docker 镜像,最终镜像体积小于 100MB。

#### 从 GitHub Container Registry 拉取

```bash
# 拉取与 git tag v1.5.7-omega 对应的镜像
# 镜像名由仓库名 Yoloccyt/Chimera-CLI- 小写并移除末尾连字符后得到
sudo docker pull ghcr.io/yoloccyt/chimera-cli:v1.5.7-omega

# 验证版本
sudo docker run --rm ghcr.io/yoloccyt/chimera-cli:v1.5.7-omega --version

sudo docker run --rm ghcr.io/yoloccyt/chimera-cli:v1.5.7-omega run "解释 OMEGA 四定律"

# 挂载配置目录(持久化 ~/.aether/omega.yaml)
sudo docker run --rm -v "$HOME/.aether:/home/nonroot/.aether" ghcr.io/yoloccyt/chimera-cli:v1.5.7-omega config show
```

> **权限说明**: 默认 Docker daemon 需要 root 或 `docker` 组成员权限。macOS/Windows Docker Desktop 无需 `sudo`。

#### 本地构建镜像

```bash
# 克隆仓库后,在仓库根目录构建本地镜像
```

### c.4 从源码构建

源码构建适用于开发者或需要自定义编译选项的场景。**编译产物 binary 名为 `aether`**,用户侧运行仍可通过 `chimela` 别名。

#### Windows 源码构建

```powershell
# 0) 克隆仓库到本地(仓库路径含空格,需用双引号包裹)
git clone https://github.com/Yoloccyt/Chimera-CLI-.git "D:\Chimera CLI"
cd "D:\Chimera CLI"

# 1) 一键配置工具链环境变量并持久化到用户级(仅需执行一次,执行后需重启终端)
.\install.ps1 -SetupEnv

# 2) 构建 release 版本(Windows GNU 工具链)
cargo build --workspace --release

# 3) 验证产物(binary 名为 aether.exe)
.\target\release\aether.exe --version
# 预期输出: aether 1.5.7-omega

# 4) 如需以 chimela 命令运行,可手动复制/重命名或运行 install.ps1 安装
#    例:复制到 PATH 目录并重命名为 chimela.exe
#    Copy-Item .\target\release\aether.exe "$env:LOCALAPPDATA\Programs\chimela\chimela.exe"
```

#### Linux / macOS 源码构建

```bash
# 0) 克隆仓库
git clone https://github.com/Yoloccyt/Chimera-CLI-.git
cd Chimera-CLI-

# 1) 安装系统依赖(Debian/Ubuntu 示例,需要 sudo)
sudo apt-get update
sudo apt-get install -y libsqlite3-dev pkg-config

# 2) 构建 release 版本
cargo build --workspace --release

# 3) 验证产物(binary 名为 aether)
```

## e. 常见问题排查

### e.1 网络 / 代理 / DNS 问题

#### 现象

- `irm` / `curl` 返回 `404 Not Found` 或 `Connection timed out`
- 脚本卡在下载阶段

#### 排查步骤

1. **确认仓库可见性**: 私有仓库必须显式传递 `Authorization: Bearer <GITHUB_TOKEN>`。
2. **检查 DNS**: 在中国大陆或某些企业网络中,`raw.githubusercontent.com` 可能被污染。可尝试:
   - 修改 DNS 为公共 DNS(如 223.5.5.5 / 8.8.8.8)
   - 使用 HTTP/HTTPS 代理
   - 直接访问 [GitHub Releases](https://github.com/Yoloccyt/Chimera-CLI-/releases) 手动下载
3. **使用代理**:

   ```powershell
   # Windows
   .\install.ps1 -Proxy 'http://proxy.company.com:8080'
   ```

   ```bash
   # Linux / macOS: 通过环境变量传递代理
   export https_proxy='http://proxy.company.com:8080'
   curl -fsSL https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.sh | sh
   ```

4. **直接克隆仓库后本地安装**: 绕过 raw.githubusercontent.com 失败。

   ```bash
   git clone https://github.com/Yoloccyt/Chimera-CLI-.git
   cd Chimera-CLI-
   export GITHUB_TOKEN='ghp_xxx'
   sh install.sh
   ```

### e.2 GITHUB_TOKEN 鉴权问题

#### 现象

- 脚本或下载返回 `404`,即使 Release 存在。
- `.deb` 或 binary 下载提示 `Not Found`。

#### 原因

私有仓库的 raw 内容与 Release 资产均拒绝匿名访问,必须提供具有 `repo` 或至少 `contents:read` 权限的 GitHub personal access token。

#### 解决

```bash
# Linux / macOS
export GITHUB_TOKEN='ghp_xxx'
curl -fsSL -H "Authorization: Bearer $GITHUB_TOKEN" \
  https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.sh | sh
```

```powershell
# Windows
$env:GITHUB_TOKEN='ghp_xxx'
$headers = @{ Authorization = "Bearer $env:GITHUB_TOKEN" }
iex (irm https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.ps1 -Headers $headers)
```

### e.3 依赖冲突 / 构建失败

#### 现象

- `cargo build` 提示找不到 `gcc` 或 linker
- Windows 下编译失败,提示 `linker not found`

#### 解决

1. **Windows**: 确保 MinGW-w64 GCC 在 PATH 中,并运行 `.\install.ps1 -SetupEnv` 配置工具链。
2. **Linux**: 安装 `libsqlite3-dev` 与 `pkg-config`:

   ```bash
   sudo apt-get update
   sudo apt-get install -y libsqlite3-dev pkg-config
   ```

3. **macOS**: 安装 Xcode Command Line Tools:

   ```bash
   xcode-select --install
   ```

### e.4 权限问题

#### 现象

- `chimela: command not found`
- 安装脚本提示 `Permission denied`
- Windows 下无法写入 `C:\Windows`

#### 解决

- **Linux / macOS**: 默认安装到 `~/.local/bin`,无需 sudo。安装后执行:

  ```bash
  export PATH="$HOME/.local/bin:$PATH"
  ```

  或重新登录 shell 以加载 `.bashrc` / `.zshrc` 更新。
- **Linux / macOS 系统级安装**: 若使用 `/usr/local/bin`,需要 `sudo`:

  ```bash
  sudo sh install.sh --install-dir /usr/local/bin
  ```

- **Windows**: 默认安装到 `%LOCALAPPDATA%\Programs\chimela`,无需管理员。若安装到系统目录,请以管理员身份运行 PowerShell。
- **macOS 安全拦截**: 若运行下载的 binary 时被 Gatekeeper 拦截,可移除隔离属性:

  ```bash
  xattr -d com.apple.quarantine ~/.local/bin/chimela
  ```

### e.5 版本不匹配

#### 现象

- `chimela --version` 输出旧版本
- 已安装新版,但系统仍调用旧版

#### 解决

1. 检查 PATH 中哪个 binary 被优先命中:

   ```bash
   which chimela
   ```

2. 若存在多个安装,删除旧版本或调整 PATH 顺序。
3. 重新打开终端,使 PATH 更新生效。

### e.6 Docker 镜像拉取失败

#### 现象

- `docker pull ghcr.io/yoloccyt/chimera-cli:v1.5.7-omega` 提示未授权或找不到

#### 解决

1. 私有仓库的 GHCR 镜像默认需要 `read:packages` 权限的 token。登录:

   ```bash
   echo $GITHUB_TOKEN | docker login ghcr.io -u <your-github-username> --password-stdin
   ```

2. 确认镜像名称已归一化: 小写并移除末尾连字符,即 `ghcr.io/yoloccyt/chimera-cli`。
3. 若仅验证功能,可本地构建镜像:

   ```bash
   docker build -t chimela-cli:local .
   docker run --rm chimela-cli:local --version
   ```
