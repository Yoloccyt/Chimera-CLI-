## 1. 选择安装方式

| 用户场景 | 推荐方式 | 依赖 |
|---|---|---|
| macOS / Linux Homebrew 用户 | `brew install chimera-cli` | Homebrew |
| Windows Scoop 用户 | `scoop install chimera-cli` | Scoop |
| Debian / Ubuntu | `.deb` + `apt` | dpkg/apt |
| Linux / macOS 普通用户 | `install.sh` | `curl` 或 `wget`、`uname` |
| Windows 普通用户 | `install.ps1` | PowerShell 5.1+ |
| 无网络 / 企业防火墙 | GitHub Release 手动下载 | 浏览器或 `curl` |
| 容器 / Kubernetes | Docker / GHCR | Docker Engine |
| 开发者 / 需要修改源码 | 源码构建 | Rust stable、MinGW (Windows) |

> **仓库可见性**: 当前仓库 `Yoloccyt/Chimera-CLI-` 为**私有仓库**。从 `raw.githubusercontent.com` 拉取脚本或从 GitHub API 下载 Release 资产时，必须在 HTTP header 中携带 `Authorization: Bearer <GITHUB_TOKEN>`。

## 2. 一键安装脚本（推荐）

### 2.1 Linux / macOS

#### 公有仓库（可省略鉴权 header）

```bash
curl -fsSL https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.sh | sh
```

或使用 `wget`：

```bash
wget -qO- https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.sh | sh
```

#### 私有仓库（必须携带 GITHUB_TOKEN）

```bash
export GITHUB_TOKEN='ghp_xxx'
curl -fsSL -H "Authorization: Bearer $GITHUB_TOKEN" \
  https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.sh | sh
```

#### 可选参数

```bash
# 指定版本
sh install.sh --version v1.5.1-omega

# 指定安装目录（默认 ~/.local/bin）
sh install.sh --install-dir /usr/local/bin

# 跳过 SHA256 校验
sh install.sh --skip-verify

# 查看帮助
sh install.sh --help
```

### 2.2 Windows

#### 公有仓库

```powershell
iex (irm https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.ps1)
```

#### 私有仓库

```powershell
$env:GITHUB_TOKEN='ghp_xxx'
$headers = @{ Authorization = "Bearer $env:GITHUB_TOKEN" }
iex (irm https://raw.githubusercontent.com/Yoloccyt/Chimera-CLI-/main/install.ps1 -Headers $headers)
```

#### 可选参数

```powershell
# 指定版本
.\install.ps1 -Version v1.5.1-omega

# 指定安装目录（默认 $env:LOCALAPPDATA\Programs\chimera）
.\install.ps1 -InstallDir C:\Tools\chimera

# 跳过 SHA256 校验
.\install.ps1 -SkipVerify

# 仅配置 Rust 工具链环境变量（源码构建场景）
.\install.ps1 -SetupEnv
```

## 3. 包管理器安装

> 以下渠道从 v1.5.1-omega 起正式提供。若您的系统未列出，请回退到第 2 节的一键脚本或第 4 节的手动下载。

### 3.1 Homebrew（macOS / Linux）

**适用场景**：已安装 Homebrew 的 macOS（Intel/Apple Silicon）或 Linux 用户。

```bash
brew tap yoloccyt/chimera-cli
brew install chimera-cli
```

**更新**：

```bash
brew update && brew upgrade chimera-cli
```

### 3.2 Scoop（Windows）

**适用场景**：已安装 Scoop 的 Windows 用户。

```powershell
scoop bucket add chimera-cli https://github.com/Yoloccyt/scoop-chimera-cli.git
scoop install chimera-cli
```

**更新**：

```powershell
scoop update chimera-cli
```

### 3.3 APT（Debian / Ubuntu）

**适用场景**：基于 Debian 的 Linux 发行版，希望使用系统包管理器安装。

从 GitHub Release 下载 `.deb` 后执行：

```bash
wget https://github.com/Yoloccyt/Chimera-CLI-/releases/download/v1.5.1-omega/chimera-cli_1.5.1-omega_amd64.deb
sudo apt install ./chimera-cli_1.5.1-omega_amd64.deb
```

**卸载**：

```bash
sudo apt remove chimera-cli
```

## 4. 从 GitHub Release 手动下载

访问 [GitHub Releases](https://github.com/Yoloccyt/Chimera-CLI-/releases) 下载对应平台资产：

| 平台 | 文件名 |
|---|---|
| Windows x86_64 | `chimera-windows-x86_64.exe` |
| Linux x86_64 | `chimera-linux-x86_64` |
| Linux aarch64 | `chimera-linux-aarch64` |
| macOS Intel | `chimera-macos-x86_64` |
| macOS Apple Silicon | `chimera-macos-aarch64` |

### 4.1 Linux / macOS 示例

```bash
# 下载（私有仓库需加 -H "Authorization: Bearer $GITHUB_TOKEN"）
curl -LO https://github.com/Yoloccyt/Chimera-CLI-/releases/download/v1.5.1-omega/chimera-linux-x86_64

# 校验 SHA256（若 Release 提供 checksums.txt）
curl -LO https://github.com/Yoloccyt/Chimera-CLI-/releases/download/v1.5.1-omega/checksums.txt
sha256sum -c checksums.txt

# 安装
chmod +x chimera-linux-x86_64
sudo mv chimera-linux-x86_64 /usr/local/bin/chimera
```

### 4.2 Windows 示例

```powershell
# 下载
Invoke-WebRequest -Uri https://github.com/Yoloccyt/Chimera-CLI-/releases/download/v1.5.1-omega/chimera-windows-x86_64.exe -OutFile chimera-windows-x86_64.exe

# 安装到用户目录（无需管理员）
$installDir = "$env:LOCALAPPDATA\Programs\chimera"
New-Item -ItemType Directory -Force -Path $installDir
Move-Item chimera-windows-x86_64.exe "$installDir\chimera.exe"

# 添加到用户 PATH
[Environment]::SetEnvironmentVariable('PATH', "$installDir;$([Environment]::GetEnvironmentVariable('PATH', 'User'))", 'User')
```

## 5. Docker 安装

镜像托管于 GitHub Container Registry (GHCR)：

```bash
# 拉取镜像
docker pull ghcr.io/yoloccyt/chimera-cli-:v1.5.1-omega

# 验证版本
docker run --rm ghcr.io/yoloccyt/chimera-cli-:v1.5.1-omega --version

# 运行单次任务
docker run --rm ghcr.io/yoloccyt/chimera-cli-:v1.5.1-omega run "解释 OMEGA 四定律"

# 挂载配置目录（持久化 ~/.aether/omega.yaml）
docker run --rm -v "$HOME/.aether:/root/.aether" ghcr.io/yoloccyt/chimera-cli-:v1.5.1-omega config show
```

> **镜像特点**: 基于 `gcr.io/distroless/cc-debian12`，无 shell，内置 `nonroot` 用户，体积 < 100MB。

## 6. 从源码构建

适用于开发者或需要自定义构建参数的场景。

### 6.1 Windows

```powershell
# 1. 克隆仓库
git clone https://github.com/Yoloccyt/Chimera-CLI-.git "D:\Chimera CLI"
cd "D:\Chimera CLI"

# 2. 配置工具链环境变量（仅需执行一次，持久化到用户环境变量）
.\install.ps1 -SetupEnv
# 执行后重新打开 PowerShell 终端

# 3. 构建
cargo build --workspace --release

# 4. 验证
.\target\release\aether.exe --version
```

### 6.2 Linux / macOS

```bash
# 1. 克隆仓库
git clone https://github.com/Yoloccyt/Chimera-CLI-.git
cd Chimera-CLI-

# 2. 构建
cargo build --workspace --release

# 3. 验证
./target/release/aether --version
```

## 7. 安装验证

无论通过哪种方式安装，完成后执行：

```bash
chimera --version
```

**预期输出**（以 v1.5.1-omega 为例）：

```text
chimera 1.5.1-omega
```

使用正则验证（与 CI 一致）：

```bash
# Linux / macOS
chimera --version | grep -qE '^(aether|chimera) [0-9]+\.[0-9]+\.[0-9]+' && echo "OK"

# Windows PowerShell
(chimera --version) -cmatch '^(aether|chimera) \d+\.\d+\.\d+'
```

基础命令测试：

```bash
chimera --help
chimera run "你好，Chimera"
```

## 8. 常见问题排查

### Q1: `chimera: command not found` / `无法将 'chimera' 项识别为 cmdlet`

**根因**: 安装目录未加入 PATH，或当前终端未刷新环境变量。

**解决**:
- Linux / macOS: 重新打开终端，或执行 `source ~/.bashrc` / `source ~/.zshrc`。
- Windows: 重新打开 PowerShell；若仍失败，检查 `Path` 用户环境变量是否包含安装目录。

### Q2: 一键脚本返回 `404 Not Found`

**根因**:
1. 私有仓库未在 HTTP header 中传递 `Authorization: Bearer <GITHUB_TOKEN>`。
2. `raw.githubusercontent.com` 被 DNS 污染或网络阻断（常见于中国大陆）。

**解决**:
- 私有仓库：严格按本文第 2 节示例传递 token。
- 网络问题：改用 GitHub Release 手动下载，或配置 DNS / 代理后重试。

### Q3: Windows 下 `Move-Item` 到 `C:\Windows` 权限不足

**根因**: `C:\Windows` 需要管理员权限。

**解决**: 将 binary 移动到用户目录（如 `$env:LOCALAPPDATA\Programs\chimera`），或以管理员身份运行 PowerShell。

### Q4: SHA256 校验失败

**根因**: 下载不完整、binary 与 checksums.txt 版本不匹配，或 Release 资产被覆盖。

**解决**:
- 重新下载 binary 和 checksums.txt。
- 确认 Release 页面资产版本与目标一致（如 v1.5.1-omega）。
- 使用 `--skip-verify` / `-SkipVerify` 临时跳过（不推荐用于生产环境）。

### Q5: Docker `--version` 无输出或镜像 > 100MB

**根因**: 镜像构建异常，或 Dockerfile 阶段缓存异常。

**解决**:
- 确认拉取的 tag 正确：`docker pull ghcr.io/yoloccyt/chimera-cli-:v1.5.1-omega`。
- 本地构建时检查 `docker build` 日志，确认使用了 `[profile.release]` 优化配置。
- 使用 `docker image inspect ghcr.io/yoloccyt/chimera-cli-:v1.5.1-omega --format '{{.Size}}'` 检查体积。

### Q6: 源码构建时 `gcc` 未找到（Windows）

**根因**: MinGW-w64 未加入 PATH。

**解决**:
```powershell
.\install.ps1 -SetupEnv
# 重新打开 PowerShell
gcc --version
```

### Q7: 私有仓库 Release 资产下载失败

**根因**: `GITHUB_TOKEN` 权限不足或过期。

**解决**:
- 确认 token 具有 `repo` 或 `read:packages` 权限。
- 检查 token 是否过期，重新生成后重试。
- 确认脚本内部读取了 `GITHUB_TOKEN` 环境变量（install.sh / install.ps1 均会读取）。
